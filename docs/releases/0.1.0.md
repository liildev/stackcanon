# StackCanon 0.1.0

## Summary

`StackCanon 0.1.0` is the first public release of the `stackcn` CLI.

It ships a stack-aware governance installer for existing JavaScript and TypeScript projects, with safe config planning, AI-facing context generation, and release-grade CLI packaging.

## Highlights

- framework coverage:
  - Next.js
  - Nuxt
  - React + Vite
  - Vue + Vite
  - TanStack Start
  - TanStack Query guidance
  - NestJS
  - Express
  - Fastify
- quality providers:
  - Ultracite
  - Biome
  - Oxlint
  - Skip
- canonical `ai/` source of truth with derived:
  - `AGENTS.md`
  - `CLAUDE.md`
  - `.ai-rulez/`
- official docs sync into `.stackcn/sources/`
- JSON contracts shared between CLI and internal web inspector
- bundled public CLI with tarball-install smoke coverage

## Main Commands

- `stackcn doctor`
- `stackcn init`
- `stackcn add`
- `stackcn generate`
- `stackcn sync`
- `stackcn revert`

## Operational Notes

- the public npm artifact is the bundled `@stackcanon/cli` package
- internal workspace packages remain private
- release PRs are driven by Changesets
- publish is handled by a separate npm trusted-publishing workflow

## Known Limits

- actual `npx ultracite init` remains a vendor-owned interactive flow
- real `npm publish` still depends on npm trusted publisher setup in the target repo
- docs sync stores normalized source content locally, but content trust review policy is still a separate concern
